# Graph NoMemory

Программа детерминированно вычисляет классический PageRank для большого невзвешенного ориентированного графа на одном узле Linux x86-64 при жестком ограничении `128 MiB` оперативной памяти. Входные данные сортируются и преобразуются порциями ограниченного размера. Ребра и каталог заданий остаются на диске, а в оперативной памяти находятся два массива оценок PageRank, исходящие степени, явно учтенные стеки потоков и ограниченные буферы. Списки входящих ребер вершин высокой степени делятся на фрагменты ограниченного размера, поэтому степень одной вершины не задает размер рабочего буфера.

PageRank выбран как глобальная метрика, вычисляемая повторными потоковыми проходами по дисковому представлению графа. Результат публикуется только после отдельного полного прохода для проверки невязки, суммарной массы и конечности всех оценок PageRank. Случайность не используется, а порядок суммирования чисел с плавающей точкой не зависит от планировщика.

Архитектура полувнешняя: объем ребер может превышать объем оперативной памяти, но состояние вершин размером `20V` байт вместе со стеками, буферами и служебным резервом не должно превышать установленное ограничение памяти. Если предварительная проверка не проходит, программа завершается до анализа. Автоматического перехода в полностью внешний режим нет.

## Сборка

Требуются CMake 3.22+, компилятор C++20, Make и POSIX Threads. Сборка проверена на Ubuntu 22.04 x86-64.

```bash
git clone https://github.com/Nikkfh5/graph_nomemory.git
cd graph_nomemory
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel
```

Сборка создает один исполняемый файл с публичным интерфейсом: `build/main`.

## Запуск: CSV → CSV

```bash
printf 'from,to\n1,2\n2,3\n3,1\n4,1\n' > edges.csv
build/main edges.csv pagerank.csv
cat pagerank.csv
```

Формат команды:

```text
main INPUT.csv OUTPUT.csv
```

`INPUT.csv` - существующий обычный файл в формате UTF-8 без BOM; последний компонент пути не должен быть символической ссылкой. Точный заголовок - `from,to`, затем по одному ребру `from -> to` на строку. Идентификатор вершины - канонический `int32` (`0|-?[1-9][0-9]*`); знак `+`, `-0`, ведущие нули, кавычки, пустые и лишние поля запрещены. Дубликаты ребер удаляются, петли сохраняются. Множество вершин образуют концы ребер; изолированные вершины задать нельзя.

`OUTPUT.csv` - еще не существующий путь внутри существующего обычного каталога. Файл создается программой, содержит `vertex,rank` и не заменяет существующий результат.

Файл конфигурации находится в [`config/main.conf`](config/main.conf). `main` автоматически читает только `./config/main.conf` относительно текущего рабочего каталога; путь в командной строке не передается. Файл содержит все изменяемые значения по умолчанию. Если файла нет, используются такие же встроенные значения.

Формат, ограничения и смысл всех ключей описаны в [`docs/configuration.md`](docs/configuration.md).

## Многопоточность и память

Основной параллельный участок - обработка заданий внутри каждого прохода PageRank со сбором вкладов по входящим ребрам. Подготовка CSV и дискового представления графа, полная проверка этого представления, суммирование векторов и публикация итогового CSV остаются последовательными.

Постоянный набор потоков `pthread` получает независимые задания без неограниченной очереди. Разные задания записывают оценки PageRank разных вершин, а частичные суммы для вершины высокой степени объединяются координатором в фиксированном порядке. Поэтому число потоков не меняет порядок суммирования чисел с плавающей точкой: в проверочных запусках результаты одного и 64 потоков совпали побитово.

`cmake --build build --parallel` распараллеливает только сборку. Число рабочих потоков программы задает `parallel.worker_count`.

Встроенный набор параметров использует `parallel.worker_count=64`, потому что воспроизводимые измерения проводились на стенде с 64 доступными логическими процессорами. Это фиксированное проверенное значение, а не скрытое автоматическое определение возможностей машины. На машине с меньшим числом доступных процессоров явно запишите результат `nproc` в файл конфигурации, чтобы не создавать лишние рабочие потоки; на более крупной машине пример сохраняет проверенный предел `64`:

```bash
workers=$(nproc)
if [ "$workers" -gt 64 ]; then workers=64; fi
sed -i "s/^parallel.worker_count=.*/parallel.worker_count=$workers/" config/main.conf
build/main edges.csv pagerank.csv
```

Фактический параллелизм ограничен минимумом из числа рабочих потоков, размера окна планировщика и числа готовых заданий, поэтому маленький граф не обязан загружать все процессоры. Масштабирование выше 64 потоков не измерялось; одного увеличения `parallel.worker_count` при `parallel.scheduler_window_records=64` для этого недостаточно.

Выбранное число потоков, стеки, защитные страницы, буферы ввода-вывода каждого потока, окно планировщика, фактический стек основного потока и резерв среды выполнения входят в предварительный расчет памяти. Запуск, не помещающийся в жесткое ограничение, отклоняется до основного прохода по ребрам. Назначение и происхождение значений параметров параллельного выполнения описаны в [`docs/configuration.md`](docs/configuration.md#как-устроен-эталонный-набор-параметров-параллельного-выполнения).

На стенде с 64 доступными логическими процессорами работали все `64` из `64` рабочих потоков; медианное ускорение вычислительного ядра составило `17.73x`. Полный этап анализа в исходном измерении ускорился только в `1.15x`, поскольку проверка и публикация CSV остаются последовательными и ограничены скоростью ввода-вывода. Условия и границы измерения приведены в [`research.md`](research.md#измерение-производительности).

## Синтетический вход

Каталог `./dataset` перед запуском должен отсутствовать: генератор не перезаписывает существующий каталог.

```bash
python3 tools/generators/generate.py \
  --profile reduced-skew \
  --output ./dataset
build/main dataset/edges.csv pagerank.csv
```

Генератор создает `edges.csv` и воспроизводимый `manifest.json`. `reduced-skew` - быстрый функциональный пример, а не демонстрация загрузки всех рабочих потоков. Профили `scale` (`1M/40M`) и `skew` (`1M/50M`, степень вершины `50000`), а также проверка результата описаны в [`tools/generators/README.md`](tools/generators/README.md); условия измерений - в [`research.md`](research.md#проверочные-графы).
